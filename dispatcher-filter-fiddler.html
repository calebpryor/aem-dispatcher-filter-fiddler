⚠ Security: Never use /glob with /type "allow". It cannot restrict by method, extension, or selectors simultaneously and can inadvertently expose admin endpoints or sensitive paths. Use specific matchers for every allow rule.
Security Insights
  • Using fewer than 2 matchers per filter is a strong sign of a poorly written, insecure rule. Bypassing one matcher is far easier than bypassing many — the more specific matchers you combine, the harder the rule is to circumvent.
  • Dispatcher filters won't stop Apache httpd from serving files directly from the document root or cache. A blocked request never reaching AEM doesn't mean a previously cached file won't be served. Always clear the cache when testing filter changes.
  • When the dispatcher blocks a request it responds with a 404 Not Found, not a 403 Forbidden — it signals the resource doesn't exist rather than that access is denied.
  • Dispatcher filters are not a substitute for a Web Application Firewall. They define which content to expose, not how to fight malicious traffic. Use ModSecurity or a cloud WAF (AWS WAF, Cloudflare, etc.) alongside your dispatcher rules for real threat protection.
